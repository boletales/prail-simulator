# prail/purs - 鉄道模型レイアウトシミュレータ

本リポジトリは、PureScript で実装された鉄道模型レイアウトシミュレータです。

---

## 1. 概要（あそびに来た人向け）

5歳の頃、夢にまで見た超巨大レイアウトが君の手に！某青い鉄道玩具のWeb版シミュレータで、君だけの鉄道王国を築きあげよう！

👉 **[Web版シミュレータを開く](https://boletales.github.io/prail-simulator/)**

### あそびかた

- **全般**：
    - 視点の移動：クリック・ドラッグ・ホイール
    - 一時停止・再開：`Space` キー
    - セーブ・ロード：「一時保存」でブラウザにセーブ、「書き出し」でファイルとしてダウンロード、「参照……」でファイルをロード、「作例読み込み」でプリセットを読み込み。
    - リセット：「作例読み込み」から「空っぽのレイアウト」を選択
- **レール**：
    - 選択：レールの端付近を左クリック。矢印キーで選択点を変更
    - 追加：`W`, `A`, `D` キーで、選択中の端点から直線・曲線左・曲線右レールを追加。その他のレールのキー割り当てについてはUI右上のタブを確認のこと。
    - 削除：端点を選択して `BackSpace`/`Delete`
    - 向きの変更：`F` で反転、`R` で回転
    - ポイント切り替え：`V`
- **列車**：
    - ⚠️ **信号機を設置していない場合は「信号に従って走行」のチェックを外さないと走りません！**
    - 追加：レールを選択して `+`
    - 列車選択：列車のいるレールを選択して `U` 、もしくは「列車操作」UIで列車を選択
    - 反転： `;` で進行方向反転
    - 削除： `*` で削除
- **信号機**：
    - 「信号に従って走行」モードでは、各列車は前方の信号機の現示（色）に従って進行します。
    - 分岐の手前・奥に信号機を設置すると、`I`, `O` キーで進路を構成できます。
    - 前方に分岐・交差がある複雑な信号機は、列車の通過・進路の変更があると赤背景（停止現示）になります。
    - 信号機ルールを設定すると、進路の構成・開通を「列車タグ」に従って自動で行います。
    - 設置・削除：`0` で信号機を設置、`9` で進入禁止標識（有効な進路と見なさない）を設置、`-` で削除
    - 進路の構成：`I`, `O` で、一つ右・左の進路を構成
    - 進路の開通：赤背景になった信号を選択して、`P` で開通
    - **信号機ルール**：
        - 「列車タグ」に従って自動でルーティングを行うためのDSLです。
        - `c` : この信号機を「複雑な信号機」として扱います。
        - `m [列車タグ] [最高速度]` : 該当する列車が通過する際に速度を制限します。
        - `o [列車タグ] [進路番号]` : 該当する列車が信号機を見つけた際、その方向に進路を開通して自動停止現示を解除します。
        - `u [列車タグ] [改変部分] [改変後]` : 該当する列車が信号機を見つけた際、列車タグを置換します。
        - `s [列車タグ]` : 該当する列車を停止させます。
        - `O [列車タグ] [進路番号]` : 該当する列車を一時停止させ、その後に `o` と同じように振る舞います。
        - `U [列車タグ] [改変部分] [改変後]` : 該当する列車を一時停止させ、その後に `u` と同じように振る舞います。
        - `r [列車タグ]` : 該当する列車を反転させます。
        - `R [列車タグ] [改変部分] [改変後]` : 該当する列車を反転させ、その後に `u` と同じように振る舞います。
        - その他の文字列はコメントとして扱われます。
        - ※ 進路番号は、右から順に `0`, `1`, `2`... です。
        - ※ 列車タグの絞り込みには正規表現を用いることができます。

## 2. 概要（読みに来た人向け）

本シミュレータは、以下の技術スタックで構成されています。詳細な構成は **[ARCHITECTURE.md](ARCHITECTURE.md)** を参照してください。    
- **Core Logic (PureScript)**: レールの敷設・接続判定、ポイントの切り替え、列車の速度・ブレーキ制御、および閉塞・連動信号制御などのシミュレーションロジック。
- **Client (HTML5 Canvas + Three.js)**: 3Dでのシミュレータ表示と、操作キー入力やコントロールパネルGUIを提供します。
- **Sync Server (Node.js + Socket.IO)**: 部屋（Room）単位でクライアント間の操作や走行状態をリアルタイムで同期し、セーブデータのファイル永続化を行います。

## 3. クイックスタート

本プロジェクトをローカル環境でビルドし、起動する手順です。

### 3.1. 必要な環境
- Node.js (v18以上推奨)
- PureScript / Spago (ビルドツール)

### 3.2. ビルド手順
プロジェクトのルートディレクトリで以下のビルドスクリプトを実行し、PureScriptコードをビルドしてJavaScriptへバンドルします。
```bash
./build.sh
```
※ 内部的には以下のコマンドが実行され、コンパイル済みのコードが `server/docs/main.js` へ出力されます。
`spago build && purs-backend-es bundle-module --no-build --to server/docs/main.js`

### 3.3. サーバーの起動
`server/` ディレクトリに移動し、Node.js サーバーを起動します。
```bash
cd server
npm install   # 初回のみ依存関係を解決
node index.mjs
```
サーバーが起動したら、ブラウザで `http://localhost:3000` にアクセスしてください。

オンラインプレイが必要ない場合、`server/docs/*` を静的に配信すれば遊べます。
